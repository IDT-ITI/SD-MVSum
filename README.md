# SD-MVSum: Script-Driven Multimodal Video Summarization Method and Datasets
This repository provides the official implementation of SD-MVSum: Script-Driven Multimodal Video Summarization Method and Datasets. It complements the release of the large datasets S-MrHiSum and S-VideoXum and includes the complete training and evaluation framework, along with all supplementary files and notes required to reproduce the experimental results. The project contains the complete SD-MVSum architecture and the configuration required to train and run the model. You can either train SD-MVSum on the provided datasets (S-VideoXum and S-MrHiSum) or use the pre-trained checkpoints (trained on S-VideoXum or S-MrHiSum) to draw immediate conclusions.

The following sections provide an overview of the datasets, with Section A detailing the structure and contents of S-MrHiSum and S-VideoXum, and Section B explaining the model implementation and usage.
## A. Datasets: [S-MrHiSum | S-VideoXum] 

The MrHiSum dataset is constructed from a curated subset of YouTube-8M videos, where highlight annotations are derived from YouTube’s “Most Replayed” statistics. These replay statistics, aggregated from at least 50 independent viewers per video, provide a reliable signal of viewer-preferred highlights. Each video is annotated at the frame level with importance scores, reflecting highlight intensity. Ground-truth video summaries are generated via a Knapsack optimization algorithm, ensuring summaries are concise while covering key highlights. In total, the dataset contains 31,892 videos with aligned visual highlight annotations, making it suitable for highlight detection and automatic video summarization tasks.

To make MrHiSum suitable for script-driven and multimodal video understanding, we further extend it into the S-MrHiSum dataset by enriching it with textual and audio modalities. Ground-truth highlights are sampled at 1 fps, and each highlighted segment is processed with LLaVA-NeXT-Video-7B to generate short textual descriptions, forming script-like annotations. In parallel, audio transcripts are extracted through a two-step pipeline: Silero is used for voice activity detection to isolate speech from background noise, and Whisper is applied for speech-to-text conversion, yielding timestamped transcripts aligned to the video timeline. The resulting dataset contains 29,918 videos, each associated with frame-level highlight labels from MrHiSum, textual descriptions of highlights, and timestamped transcripts aligned with audio.

The S-VideoXum dataset is an extension of the original VideoXum benchmark and was previously introduced in earlier work, where visual highlights were paired with textual scripts generated by LLaVA-NeXT. Enhancement of this dataset is achieved by incorporating aligned audio transcripts, thereby adding a new modality for multimodal video understanding. Segments containing speech are detected using Silero VAD, and transcriptions are obtained with Whisper ASR, resulting in timestamped transcripts that are temporally aligned with the video content. This extension complements the existing textual descriptions with temporally aligned transcripts, offering richer multimodal supervision and enabling analysis that integrates visual, textual, and audio signals.

The CLIP embeddings (visual, textual, and transcript) included with the datasets were extracted using the fine-tuned CLIP model: visual embeddings from videos downsampled at 1 fps, textual embeddings from scripts split into sentences, and transcript embeddings from timestamped audio transcripts parsed, chunked, and aligned with their timestamps.

---
### 1.1 `script_mrhisum.h5`
The core HDF5 file for the S-MrHiSum dataset. Each top-level group corresponds to one video, named by its video_name. Inside each video group:
 
| Key                   | Description                                                                                                      | Shape / Type                                |
|-----------------------|------------------------------------------------------------------------------------------------------------------|---------------------------------------------|
| `n_frames`            | Number of frames in the original video.                                                                          | Scalar integer                              |
| `change_points`       | Shot boundaries, start–end frame indices.                                                                        | `[num_shots, 2]`                            |
| `gt_summary`          | Binary ground-truth summary derived from gtscores using the knapsack algorithm with a 15% budget.                 | `[n_downsampled_frames]` (binary vector)    |
| `video_embeddings`    | Frame-level CLIP embeddings for sampled frames at 1 fps.                                                         | `[num_downsampled_frames, 512]`             |
| `text_embeddings`     | CLIP feature tensor for the scripts.                                                                             | `[M, 512]` (M = number of sentences)        |
| `transcript_embeddings` | CLIP embeddings for chunks of the audio transcript.                                                            | `[N, 512]` (N = number of transcript chunks) |
| `transcript_timestamps` | Start and end times for each transcript chunk, aligned with transcript_embeddings.                             | `[N, 2]`                                    |
| `aligned_transcripts` | Transcript embeddings aligned to the sampled video frames, with zero-padding where no transcript is available.   | `[num_downsampled_frames, 512]`             |




### 1.2 `script_videoxum.h5`
The core HDF5 file for the S-VideoXum dataset. Each top-level group corresponds to one video, named by its video_name. Inside each video group:

| Key                    | Description                                                                                                         | Shape / Type                        |
|------------------------|---------------------------------------------------------------------------------------------------------------------|-------------------------------------|
| `n_frames`             | Number of downsampled frames in the video.                                                                          | Scalar integer                       |
| `gtscores`             | Ground‐truth scores from the 10 human annotators.                                                                   | `[10, n_frames]`                    |
| `video_embeddings`     | CLIP feature matrix for each frame.                                                                                 | `[n_frames, 512]`                   |
| `text_embeddings`      | CLIP feature tensor for the 10 scripts (one per annotator). Zero-padded if shorter than `M_max`.                     | `[10, M_max, 512]`                   |
| `transcript_embeddings`| CLIP embeddings for chunks of the audio transcript.                                                                 | `[N, 512]` (N = number of transcript chunks)    |
| `transcript_timestamps`| Start and end times for each transcript chunk, aligned with transcript_embeddings.                                   | `[N, 2]`                            |
| `aligned_transcripts`  | Transcript embeddings aligned to the sampled video frames (video_embeddings), zero-padded where no transcript exists. | `[n_frames, 512]`                   |

### 3. Text Annotations

This folder, available on Zenodo, contains all textual data used in the datasets, including generated scripts and timestamped audio transcripts. The scripts were generated using LLaVA-NeXT-Video-7B, and transcripts were extracted using Silero and Whisper.

#### 3.1 `Scripts/`
  Contains the generated scripts for MrHiSum videos, produced by LLaVA-NeXT-Video-7B.

#### 3.2 `Transcripts/`
  Contains timestamped audio transcripts for both datasets, organized into subfolders produced by Whisper Turbo:
    
      `S-MrHiSum/` — transcripts for MrHiSum videos
        
      `S-VideoXum/` — transcripts for VideoXum videos

## B. SD-MVSum method and models

This repository implements SD-MVSum, a method for script-driven video summarization, and provides scripts to train, test, and run the model in inference.

### Installation

Clone the repository
   ```
  git clone https://github.com/IDT-ITI/SD-MVSum.git
  cd SD-MVSum
   ```
Create and activate the Conda environment
   ```
   conda env create -f environment.yml
   conda activate sd_mvsum
   ```
### Dataset Preparation

Download the datasets
   - Retrieve the .h5 files and the split.json files from Zenodo.
   - Place both files under the ```dataset``` directory
      ```
      SD-MVSum
       └── dataset/
            ├── script_mrhisum.h5
            ├── script_mrhisum_split.json
            ├── script_videoxum.h5
            └── script_videoxum_split.json
      ``` 

### Pretrained Models

Download the pretrained SD-MVSum models trained on S-VideoXum and S-MrHiSum from the Zenodo directory, and save them to a local folder.

### Usage
SD-MVSum can be employed for both training on the published datasets and conducting inference using pretrained models.

#### Train model on S-VideoXum and S-MrHiSum
Both S-VideoXum and S-MrHiSum include train, validation, and test splits. After each training epoch, the model is evaluated on the validation set, and once training is complete, the best-performing checkpoint on the validation set is evaluated on the test set. The final model is saved as a .pkl file. To train on either dataset, run one of the following commands:
```
python main.py --exp_num='exp1' --epochs=E --batch_size=B --train=True --dataset='S_VideoXum'
python main.py --exp_num='exp2' --epochs=E --batch_size=B --train=True --dataset='S_MrHisum'
```


#### Inference on Pretrained Models
To run inference using a pretrained checkpoint on either S-VideoXum or S-MrHiSum:

```
python main.py --exp_num='exp1' --ckpt_path='path/to/pkl/file' --train=False --dataset='S_VideoXum'
python main.py --exp_num='exp2' --ckpt_path='path/to/pkl/file' --train=False --dataset='S_MrHisum'
```
